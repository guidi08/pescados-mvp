# Checklist de lançamento (30 dias)

## Semana 1 — Infra e base

- [ ] Supabase criado + SQL aplicado + RLS testado
- [ ] Stripe configurado (Connect + Webhook)
- [ ] API em produção + webhook recebendo eventos
- [ ] Portal no ar (Vercel)
- [ ] App rodando em dev (Expo)

## Semana 2 — Fornecedores

- [ ] 2–5 fornecedores onboarded
- [ ] Produtos cadastrados + variantes
- [ ] E-mail de pedido funcionando (SMTP)
- [ ] Testes de pedido real em ambiente de teste

## Semana 3 — Operação real controlada

- [ ] Piloto com poucos compradores (B2B)
- [ ] Regras de cancelamento validadas
- [ ] Ajuste por peso testado (salmão)

## Semana 4 — Publicação e escala leve

- [ ] Publicar Android (Internal → Production)
- [ ] Publicar iOS (TestFlight → Production)
- [ ] Monitorar chargebacks e reservas
- [ ] Ajustar taxas / reserva conforme necessidade
